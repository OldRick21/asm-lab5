# Лабораторная работа №5 «Разработка функций на языке Ассемблера и встраивание их в программы на языке C»

## Введение
**Цель работы:** изучение взаимодействия кода на языке C и Ассемблера.  
Данный аспект важен для создания переносимых программ с минимальными временными затратами на разработку.

### Этапы разработки ПО:
1. Проектирование программы
2. Разработка программы
3. Отладка программы
4. Анализ производительности
5. Оптимизация программы

Принцип Kента Бека:  
*«Make it work, make it right, make it fast»*

**Ключевая стратегия:** оптимизация на Ассемблере только наиболее ресурсоемких участков кода.

---

## Требования к выполнению
1. Разработать программу на C с требуемой функциональностью
2. Провести анализ производительности для режимов оптимизации:
   - `-O0`
   - `-O1`
   - `-O2`
   - `-O3`
   - `-Ofast`
3. Реализовать основную функцию обработки изображения на Ассемблере
4. Провести повторный анализ производительности

## Примечания
1. **Аргументы командной строки:**  
   Имена входного/выходного файлов передаются через аргументы.
2. **Проверки:**  
   Программа должна проверять:
   - Наличие файлов
   - Корректность данных
   - Валидность параметров
3. **Структура программы на C:**  
   - Основная функция обработки изображения должна работать с буфером
   - Запрещено использование библиотечных функций внутри этой функции
4. **Реализация на Ассемблере:**
   - Код размещается в отдельном файле `.s`
   - Для работы с изображениями разрешено использовать готовые C-библиотеки
5. **Анализ производительности:**  
   Результаты представить в виде таблицы с тестами на разных примерах.
